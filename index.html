Flagship Project

Local-first LLM automation platform built around explicit authority boundaries.

This project matters because it does not present RAG as a toy chatbot. It treats retrieval, chat, document ingestion, and model access as a product architecture problem.

  • Architecture: Next.js UI, Python backend API, Python worker process, local Ollama runtime
  • Retrieval model: hybrid SQLite FTS plus workspace-local LanceDB vectors
  • Ingestion model: immutable document versions, parse artifacts, chunk manifests, embeddings, and scoped retrieval
  • Security posture: loopback-only services, no browser-side model access, local-only policy gates, validated structured outputs
  • Implemented now: authenticated uploads, chunking, workspace-isolated vector storage, structured citation-aware chat streaming
