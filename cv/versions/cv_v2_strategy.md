# CV Rewrite Strategy

## 1. New positioning statement

Junior full-stack developer building local-first AI systems, document pipelines, and workflow automation for real operational use. Production experience spans Python backends, Flask and Node.js web apps, retrieval-driven AI workflows, and deployment support across Linux-hosted environments.

## 2. Target persona

`AI-augmented Full Stack Developer focused on automation, RAG systems, and production workflows`

## 3. CV structure redesign

### New section order

1. Header and positioning
2. Technical stack
3. Selected projects
4. Experience
5. Education
6. Languages

### What to remove

- The separate `Core Strengths` section
- Most of the standalone training list
- Repetitive summary language
- Low-signal projects that do not strengthen the target narrative

### What to compress

- Outlier experience
- Generic tooling lists
- Broad claims like `production ownership` when the project bullets can prove it better

### What to expand

- The local-first RAG platform
- High-signal production systems tied to automation and backend engineering
- Full-stack evidence where you owned both application code and deployment surfaces
- Security, retrieval, validation, and orchestration decisions

## 4. Core themes

- Local-first AI systems with explicit architecture and security boundaries
- Document-heavy workflow automation for finance, insurance, and accounting operations
- Full-stack delivery that includes backend logic, user-facing flows, and production support

## 5. Differentiation angle

You are not strongest when presented as a generic junior developer who knows many tools. You stand out when framed as a builder of systems that turn messy, high-friction operational inputs into auditable software outputs. That is a more serious and memorable story.

## 6. Project prioritization

### Rank 1. Local-First LLM Automation Platform

Why it leads:

- Best match for target roles in AI, RAG, SaaS, and modern full-stack teams
- Shows architecture judgment, security constraints, retrieval design, and backend orchestration
- Lets you surface Next.js, Python backend design, Ollama integration, vector storage, chunking, structured outputs, and chat streaming naturally

How to position it truthfully:

- `Architecting and building`, not `deployed at scale`
- Distinguish implemented pieces from target architecture
- Emphasize production-oriented design rather than pretending the whole platform is finished

### Rank 2. Portfolio PDF to Excel Automation Platform

Why it stays high:

- Strongest proof of document processing, OCR fallback, CLI automation, and auditable output design
- Demonstrates systems thinking on messy business inputs

### Rank 3. Insurance Premium Automation Platform

Why it matters:

- Best proof of production backend ownership
- Shows API integration, validation, normalization, and business-rule complexity

### Rank 4. Client Acquisition and Flask Portal Pipeline

Why it stays:

- Strongest full-stack and deployment proof
- Shows end-to-end ownership across user-facing web flows and infrastructure

### Rank 5. Cashflow Desktop Application

Why it stays if space allows:

- Good evidence that you can productize automation into a usable client application
- Shows responsiveness, packaging, and business-focused UX decisions
